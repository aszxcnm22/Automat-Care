# เอกสารการอัปเดตระบบ

## Ticket & Access Management System

เอกสารนี้อธิบายรายละเอียดการปรับปรุงและพัฒนาระบบ **Ticket & Access Management System** โดยครอบคลุมการจัดการไฟล์อัปโหลด ระบบจัดการโปรเจกต์ การควบคุมสิทธิ์ผู้ใช้งานตาม Role และการตรวจสอบข้อมูลซ้ำภายในระบบ

---

# สารบัญ

1. ระบบการจัดการไฟล์ (File Upload Management)
2. ระบบจัดการโปรเจกต์ (Project Management System)
3. การจัดการสิทธิ์การมองเห็นเมนู (Role-based Menu Visibility)
4. การปรับหัวข้อหน้าจอแบบไดนามิก (Dynamic Page Title)
5. Mock Users สำหรับการทดสอบระบบ
6. การปรับปรุงหน้า Projects
7. การเพิ่ม Role ใหม่ GUEST
8. การจำกัดการมองเห็นเมนูสำหรับ Guest
9. การซ่อนปุ่ม View Tickets สำหรับ Guest
10. การแสดงข้อมูลสำหรับผู้ใช้ Guest
11. การตรวจสอบข้อมูลซ้ำในหน้า Add New User
12. การตรวจสอบข้อมูลซ้ำในหน้า Add New Group
13. ระบบจัดการสถานะข้อผิดพลาด (Error State Management)

---

# 1. ระบบการจัดการไฟล์ (File Upload Management)

**ไฟล์ที่แก้ไข**

```
src/pages/Tickets.tsx
```

## การตรวจสอบประเภทไฟล์ที่รองรับ

ระบบทำการตรวจสอบประเภทไฟล์ภายในฟังก์ชัน

```
handleFileChange
```

ประเภทไฟล์ที่ระบบรองรับ

* JPG
* PNG
* PDF
* TXT
* LOG
* ZIP
* XLSX
* CSV

โดยเฉพาะไฟล์ประเภท

```
.xlsx
.csv
```

ซึ่งได้รับการยืนยันว่าระบบรองรับการใช้งานได้ตามปกติ

---

## การจำกัดขนาดไฟล์

ระบบกำหนดขนาดไฟล์สูงสุดสำหรับการอัปโหลด

```
ไม่เกิน 10MB ต่อไฟล์
```

หากไฟล์มีขนาดเกินกว่าที่กำหนด ระบบจะไม่อนุญาตให้อัปโหลดไฟล์ดังกล่าว

---

## การปรับปรุงข้อความแจ้งเตือนใน UI

ตำแหน่ง

```
Report New Issue
```

ได้มีการปรับปรุงข้อความคำแนะนำในการอัปโหลดไฟล์ เพื่อให้ผู้ใช้งานเข้าใจประเภทไฟล์ที่รองรับได้อย่างชัดเจน

ตัวอย่างข้อความที่แสดงในระบบ

```
Supported file types: JPG, PNG, PDF, TXT, LOG, ZIP, XLSX, CSV
Maximum file size: 10MB
```

---

## การดาวน์โหลดไฟล์แนบ

ไฟล์ที่ถูกแนบใน Ticket สามารถดาวน์โหลดได้ผ่านปุ่ม

```
Download
```

ในหน้าจอ

```
View Ticket
```

# 2. ระบบจัดการโปรเจกต์ (Project Management System)

```
src/pages/Projects.tsx
```

## ฟังก์ชันหลัก

| ฟังก์ชัน         | รายละเอียด                             |
| ---------------- | -------------------------------------- |
| Projects         | โครงสร้างหน้าหลักของระบบจัดการโปรเจกต์ |
| handleOpenModal  | เปิดหน้าต่าง Popup สำหรับฟอร์ม         |
| handleCloseModal | ปิดหน้าต่าง Popup                      |
| handleSubmit     | ใช้สำหรับสร้างหรือแก้ไขโปรเจกต์        |
| handleDelete     | ใช้สำหรับลบโปรเจกต์                    |

---

## การเพิ่มระบบป้องกันการเข้าถึง (Route Guard)

ระบบอนุญาตให้เฉพาะผู้ใช้ที่มี Role เป็น **Admin** เท่านั้นที่สามารถเข้าถึงหน้า Projects ได้

เงื่อนไขที่ใช้ตรวจสอบ

```
user.role !== 'Admin'
```

หากผู้ใช้ไม่มีสิทธิ์ ระบบจะแสดงหน้า

```
Access Denied
```

# 3. การจัดการสิทธิ์การมองเห็นเมนู (Role-based Menu Visibility)

**ไฟล์ที่แก้ไข**

```
src/components/Layout.tsx
```

## เมนู Projects

เมนูนี้จะแสดงเฉพาะผู้ใช้ที่มี Role เป็น **Admin**

```
{user?.role === 'Admin' && (...)}
```

ใช้กับ

* Sidebar (Desktop)
* Mobile Menu

---

## เมนู Access Management แบบ Dynamic

ระบบจะปรับชื่อเมนูตาม Role ของผู้ใช้งาน

```
{user?.role === 'Admin'
  ? 'Access Management'
  : user?.group || 'Access Management'}
```

# 11. การตรวจสอบข้อมูลซ้ำในหน้า Add New User

**ไฟล์**

```
src/pages/AccessManagement.tsx
```

### ฟังก์ชัน

```
handleAddUser
```

### การปรับปรุง

เปลี่ยนวิธีการแจ้งเตือนจาก

```
alert()
```

เป็นการจัดการผ่าน **State**

เพื่อให้ข้อความแจ้งเตือนแสดงผลภายในฟอร์ม และเหมาะสมกับ UX ของระบบ

---

## State ที่เพิ่ม

```
addUserError
```

ใช้สำหรับเก็บข้อความแจ้งเตือนเมื่อพบ

* Username ซ้ำ
* Email ซ้ำ

---

## การปรับปรุง UI

ภายใน

```
Add New User Modal
```

ระบบจะแสดงข้อความ Error

ลักษณะการแสดงผล

* สีแดง
* อยู่ภายในฟอร์ม

ตัวอย่างข้อความ

```
Email already exists in the system.
```

หรือ

```
Username already exists.
```

## ตรรกะการตรวจสอบข้อมูล

ระบบใช้

```
groups.some(...)
```

ร่วมกับ

```
.toLowerCase()
```

เพื่อเปรียบเทียบข้อมูลแบบ **ไม่สนใจตัวพิมพ์เล็ก–ใหญ่**

---

## เงื่อนไขการตรวจสอบ

ระบบจะตรวจสอบ

* Group Name ซ้ำ
* Email Domain ซ้ำ

หากพบข้อมูลซ้ำ

```
setAddGroupError()
return
```

ฟังก์ชันจะหยุดทำงานทันที

---

# 13. ระบบจัดการสถานะข้อผิดพลาด (Error State Management)

เพิ่ม State ใหม่

```
const [addGroupError, setAddGroupError] = useState(null);
```

ใช้สำหรับ

* เก็บข้อความ Error
* ควบคุมการแสดงผล Error ใน UI

---

## การปรับปรุงปุ่ม Close ใน Modal

เมื่อผู้ใช้กดปิดหน้าต่าง

```
setAddGroupError(null)
```

ระบบจะทำการล้างค่า Error ก่อนปิดหน้าต่าง Modal

---

# สรุปการอัปเดตระบบ

การอัปเดตครั้งนี้ประกอบด้วยการปรับปรุงสำคัญดังต่อไปนี้

* เพิ่มระบบ **จำกัดประเภทและขนาดไฟล์อัปโหลด**
* เพิ่ม **ระบบจัดการโปรเจกต์**
* เพิ่ม **การควบคุมสิทธิ์ผู้ใช้งานตาม Role**
* เพิ่ม **Role ใหม่ GUEST**
* เพิ่ม **ระบบตรวจสอบข้อมูลซ้ำ**
* ปรับปรุง **Error Handling และ UX**
* เพิ่ม **Dynamic UI ตาม Role ของผู้ใช้**

การปรับปรุงเหล่านี้ช่วยให้ระบบมีความ **ปลอดภัย ใช้งานง่าย และรองรับการขยายระบบในอนาคตได้ดีขึ้น**
